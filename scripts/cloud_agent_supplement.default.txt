<cloud_task_instructions>
Cloud Agents that work on GitHub issues and pull requests typically receive task rules in addition to the desktop Cursor system prompt. Common themes include:

- Work only on the task-assigned development branch; treat the specified base branch (often main) as merge target unless told otherwise. Do not push to unrelated branches without explicit instruction.
- Keep commits small and descriptive; push regularly. When pushing, use `git push -u origin <branch-name>`. If network errors occur, retry with bounded exponential backoff a few times.
- When syncing, prefer fetching or pulling named branches rather than unspecified defaults.
- At the end of a work iteration, if the product provides a pull-request tool, create or update the PR to match the latest pushed revision; use the repository PR template for the body when one exists; draft PRs are often the default unless the user prefers otherwise.
- Before opening or updating a PR, ensure changes are committed and pushed.
</cloud_task_instructions>
